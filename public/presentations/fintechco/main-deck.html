FinTechCo × Claude Code03 · Discovery
Principles before products

Zero trust, extended to agents: least agency

Verify every actor

Humans, services, and now agents authenticate the same way: strong identity, phishing-resistant MFA, no standing credentials. An agent is a principal, not a plugin.

Least privilege, finer grained

Least agency is least privilege applied at the level of individual tools and actions: an agent gets exactly the capabilities a task needs, for exactly as long as it needs them.

Assume compromise

Design so that a hijacked prompt or poisoned context is contained by scope, not by hope: bounded tools, reviewed actions, audited everything.

This is the bar regulators and federal frameworks are converging on. The honest tension: zero trust trades a little speed for a lot of safety. The rest of this briefing shows how you keep the productivity win anyway.

Appendix A2

Claude Code security pro-tips: determinism where it counts

Hooks: policy as code

PreToolUse hooks run before any tool call: block writes to protected paths, require passing tests or lint before sensitive actions, verify signatures on artifacts.

PostToolUse / session hooks feed audit pipelines, so review happens on evidence rather than memory.

Scoped capability

Allowed-tools and permission modes define exactly what an agent may do in each repo; a sandbox area and a production-like area carry different scopes.

CLAUDE.md turns team conventions into instructions the agent follows on every run.

Managed at the org, not the laptop

Centrally managed policy settings push the same security configuration to all 180 engineers; individual machines cannot quietly drift below the bar.

Observable by default

Telemetry export (OpenTelemetry-compatible) into your existing SIEM and dashboards: usage, actions, and spend, the same way you watch any other production system.

Appendix A3

Just-in-time access: the Enterprise-tier keystone

1

No standing credentials

Agents and engineers request access scoped to a task; grants are short-lived and expire on their own. A leaked credential that dies in minutes is a contained event, not an incident.

2

Approval matched to blast radius

Low-risk scopes auto-approve against policy; sensitive scopes require a named human approver. The approval path is the policy, written down and enforced.

3

Every grant audited

Who asked, what for, who approved, what was done with it: one queryable trail that turns regulator questions into lookups instead of investigations.

4

Phased adoption

Start with the agent’s credentials in the pilot (smallest blast radius, newest principal type), then extend the same machinery to service and human identities on the Enterprise timeline.

Deep dive: GitHub Copilot and Cursor

GitHub Copilot

Strengths: enormous install base, frictionless inside the GitHub estate, strong autocomplete, familiar procurement.

Limits for FinTechCo: shallower autonomous depth on multi-file, cross-service tasks; security configuration is broader-brush than hook-level policy enforcement; agent behaviors are newer and less battle-tested.

Where it wins: if the goal were marginal autocomplete lift only, Copilot is the path of least resistance.

Cursor

Strengths: excellent in-editor agent UX, fast iteration, popular with individual power users; routes across multiple model providers.

Limits for FinTechCo: an IDE fork to standardize and patch across 180 people; governance surface is editor-scoped; multi-provider routing complicates the data-flow story your regulators will ask about.

Where it wins: individual developer delight, especially for engineers who live entirely in the editor.

Appendix A10

Five threats every agentic deployment must answer for

1

Prompt injection

Untrusted content steering the agent’s actions.

2

Tool hijacking

Abusing the tools and resources an agent can reach.

3

Privilege abuse

Identity and access misuse through the agent’s permissions.

4

Context poisoning

Corrupting memory or context the agent relies on.

5

Supply chain

Compromised packages, models, or extensions upstream.

These surfaces are months old and still moving. The right response is not a frozen checklist; it is a security culture that expects the attack surface to evolve as fast as the technology underneath it, and re-verifies its controls on that cadence.
